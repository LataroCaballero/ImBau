<objective>
Stand up the shared observability foundation that plan 04-02 wires into the three apps: install the pinned Sentry SDKs + pino stack, create a shared `@imbau/observability` logger built on the pino-loki transport, and add the `sentryEnv` + `lokiEnv` env presets and the Sentry build-time passthrough to `turbo.json`.

Purpose: OBS-01 (Sentry) and OBS-02 (pino → Loki) both need a single, RAM-optimal, fallback-symmetric logging path and a typed env contract before any app code can be instrumented. Centralizing the pino+pino-loki logger avoids 3× duplication and guarantees the same transport targets local Loki OR Grafana Cloud by swapping `LOKI_URL` (D-04).
Output: a new `packages/observability` package, extended env presets, updated `turbo.json`, and the app deps installed (no app source changes yet — that is 04-02).
</objective>

<phase_decisions>
## Phase-4 decisions adopted (RESEARCH §Open Questions) — documented here as the phase entry plan

1. **pino-loki transport over Promtail (refines D-03).** D-03 names "Promtail"; we use the in-app **pino-loki** transport instead. Rationale: zero extra container (~0 local RAM vs ~64 MB), and the SAME transport points at local `loki:3100` OR Grafana Cloud by swapping `LOKI_URL`/`LOKI_BASIC_AUTH` — symmetric with the mandatory D-04 fallback. This is a refinement of D-03's intent ("pino → Loki"), not a contradiction; Promtail/Alloy remains a documented alternative.
2. **Postgres backups → DEFERRED** (FOUND-01, CONTEXT Deferred Ideas). Staging DB is ImBau-owned, ephemeral, reconstructible from migrations (+future seed). Named volumes persist restarts; no pgBackRest/wal-g this phase.
3. **Edge rate-limit for events/leads → DEFERRED.** Those tables are a future milestone; the nginx `limit_req` mechanism is noted in plan 04-05 for when they land.
4. **Grafana / Uptime-Kuma → NOT publicly exposed** (least attack surface). Internal docker network; access via SSH tunnel; Grafana anonymous auth off. See plans 04-05/04-07.
5. **Two subdomains + one SAN cert.** `staging.tours.andescode.com.ar` (web) + `panel.staging.tours.andescode.com.ar` (panel), one SAN cert. ⚠ This requires **TWO DNS A records**, not one — flagged in plans 04-05 and 04-07 prerequisites.
