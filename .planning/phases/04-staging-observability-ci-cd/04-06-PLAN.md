<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → GHCR | Image push authenticated by GITHUB_TOKEN (packages:write) |
| GitHub Actions → VPS | SSH deploy with a private key secret |
| deploy.sh → staging DB | Owner-role DDL + role-password provisioning |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-4-CRED | Spoofing/Tampering | GHCR token, VPS_SSH_KEY, age key | high | mitigate | GITHUB_TOKEN scoped packages:write; VPS_SSH_KEY/SOPS_AGE_KEY as Actions secrets; VPS pulls with a PAT scoped read:packages; rotate on leak |
| T-4-AUTH-PW | Spoofing | staging app/anon DB auth | high | mitigate | bootstrap-roles.sql provisions real passwords from SOPS (Pattern 4); passwords passed via psql `-v`, never logged |
| T-4-MIGRATE | Tampering/Availability | schema migration on deploy | high | mitigate | migrate-before-swap one-off container; non-zero exit aborts via `set -e` before any app swap (old containers stay up) |
| T-4-SECRET | Information Disclosure | decrypted runtime .env | medium | mitigate | `.env` chmod 600, gitignored; no secret echoed in deploy logs |
</threat_model>
