<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → withTenant/withAnon | The only sanctioned data path; the `orgId` argument crosses into the SQL layer and must be parameterized, not interpolated. |
| pooled connection → request | A reused pooled connection could carry a prior request's tenant GUC if it were session-scoped. |
| test runner → DB role | The absence test is meaningless unless it runs as the unprivileged app/anon role. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Information Disclosure | cross-tenant data read (broken/absent policy) | mitigate | Absence tests assert ZERO rows of the other org as the app role; this is the exit gate. Tenant policy + FORCE RLS from plan 02. |
| T-02-11 | Tampering | SQL injection via tenant-context path | mitigate | `set_config('app.current_organization_id', ${orgId}, true)` parameterized bind; never string-interpolated into `SET LOCAL` (verified by grep gate + code review). |
| T-02-12 | Information Disclosure | pooled-connection GUC bleed across requests | mitigate | GUC set only inside `db.transaction()` with `is_local=true` (auto-clears at txn end); never session `SET`. |
| T-02-13 | Tampering | cross-tenant write | mitigate | `withCheck` on the tenant policy (plan 02); case (c) asserts cross-tenant INSERT/UPDATE fails. |
| T-02-14 | Elevation of Privilege | absence test passes for the wrong reason (privileged role) | mitigate | Setup guard asserts `rolbypassrls=false`/`rolsuper=false` and the expected `current_user`; assertions never run as owner. |
| T-02-15 | Information Disclosure | draft leaks via anon path | mitigate | Case (d) asserts anon sees zero `borrador`; relies on plan-02 anon SELECT-only published policy. |
</threat_model>
